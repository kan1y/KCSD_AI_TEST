База знаний бизнес-процессов АО "Центральный Бомбарий"
Описание проекта
Данный репозиторий представляет собой единую, строго структурированную и машиночитаемую базу знаний бизнес-процессов. Архитектура проекта разработана специально для интеграции с ИИ-агентами (оркестраторами) и удобного управления процессами со стороны  (ОБА).

Все артефакты нормализованы и разделены по смысловым директориям. Это позволяет ИИ-агенту точечно извлекать нужный контекст (алгоритмы, метрики, графические схемы) без перегрузки памяти, связывая данные по уникальному идентификатору (по принципу реляционных баз данных).

Связывание данных (Relational Key)
Ключевым паттерном репозитория является использование уникального идентификатора процесса (например, M_8_1, B_2_2, S_1_2).

Чтобы получить полную картину по конкретному процессу, ИИ-агент или аналитик должен собрать данные из файлов с одинаковым именем (например, M_8_1.*) по всем рабочим директориям репозитория.

Структура репозитория
00-AGENT-INSTRUCTIONS.md — системный промпт и правила контекста для ИИ-агента.

10-Governance/ — нормативно-справочная база:

Правила описания бизнес-процессов.

Порядок ведения Базы знаний в YouTrack.

21-Process-Cards/ — паспорта процессов: классификация, владельцы, участники, входы и выходы, используемые ИС.

22-Processes/ — текстовые пошаговые алгоритмы выполнения процессов (Execution flow).

23-Scoring/ — модели скоринга для оценки критичности, ценности и приоритизации задач на оптимизацию/автоматизацию.

24-Metrics/ — операционные метрики процессов (Cycle Time, Processing Time, Wait Time, Error Rate, Rework, Volume).

25-BPMN-Models/ — исходные XML-файлы графических моделей в нотации BPMN 2.0 (исполняемые, подготовленные в Camunda Modeler 8).

Инструкция по добавлению нового процесса
При добавлении нового бизнес-процесса в базу знаний необходимо соблюдать принцип ссылочной целостности. Для нового процесса (например, B 2.2) необходимо создать 5 файлов с идентичным именем в соответствующих директориях:

21-Process-Cards/B_2_2.md

22-Processes/B_2_2.md

23-Scoring/B_2_2.md

24-Metrics/B_2_2.md

25-BPMN-Models/B_2_2.bpmn

Внутри текстовых .md файлов обязательно указывать тег ID: B 2.2 для дополнительного контроля целостности данных при парсинге.

Используемые технологии и интеграции
Git — версионирование бизнес-логики, текстовых артефактов и моделей.

Camunda Modeler 8 — проектирование, валидация и исполнение BPMN-схем.

YouTrack — ведение пользовательской базы знаний, планирование спринтов и управление задачами по оптимизации процессов.

LLM / AI Agents — автоматизированный аудит, поиск узких мест (bottlenecks) и проверка процессов на соответствие нормативной базе (compliance).
